# Free-tier deployment (Render + optional Netlify)

You cannot finish cloud sign-in from this repo alone: create accounts on the host(s) below and connect **GitHub → your fork or `HarryRobertL/JOB-flow`**.

## Option A — Recommended: one URL on Render (no Netlify)

The Docker image builds the React app into `dist/` and FastAPI serves **API + SPA on the same origin**, so you do **not** need CORS or `VITE_API_BASE_URL`.

1. Sign up at [Render](https://render.com) (GitHub login is fine).
2. **New → Blueprint** → select this repository → Render reads [`render.yaml`](../render.yaml).
3. Wait for the first deploy. `releaseCommand` runs [`seed_users.py`](../seed_users.py) so demo users exist.
4. Open the service **URL** (e.g. `https://jobflow.onrender.com`). Use `/health` to verify.
5. Copy that URL into the **Live demo** table in [`README.md`](../README.md).

**Notes (free tier):** The instance **spins down** after idle time; the first request can take ~30–60s. SQLite and `data/` are **ephemeral** unless you add a paid disk — fine for demos; users reset on redeploy.

**Env vars** (Blueprint already sets):

| Variable | Purpose |
|----------|---------|
| `AUTOAPPLYER_SESSION_SECRET` | Generated by Render |
| `AUTOAPPLYER_SECURE_COOKIES` | `true` (HTTPS) |

Optional: set `AUTOAPPLYER_SESSION_SAMESITE=none` only if you also use Option B from the same API.

## Option B — Netlify (static) + Render (API)

Use this if you want the SPA on **Netlify** and the API on **Render**.

1. Deploy the **same Docker service** on Render as in Option A. Note the API origin, e.g. `https://jobflow-xxxx.onrender.com` (no trailing slash).
2. In **Render → Environment**, add:

   `AUTOAPPLYER_CORS_ORIGINS=https://YOUR-SITE.netlify.app`

   Use the exact Netlify URL (scheme + host, no path). For multiple origins, comma-separate.

3. **Redeploy** the Render service so CORS and `SameSite=None` session cookies apply.
4. In **Netlify**: New site from Git → build command `npm run build`, publish `dist/`.
5. **Netlify → Site settings → Environment variables**: set `VITE_API_BASE_URL` to your Render URL (no trailing slash). Trigger a new deploy.
6. Update [`README.md`](../README.md) **Live demo** with the **Netlify** URL (that is what visitors open).

The app sets **cross-site cookies** when `AUTOAPPLYER_CORS_ORIGINS` is non-empty (`SameSite=None; Secure`).

---

## Option C — Cloudflare Pages (static) + Render (API)

Same as Option B: set `VITE_API_BASE_URL` in Pages build env and `AUTOAPPLYER_CORS_ORIGINS` to your Pages URL (`https://….pages.dev`).

## Troubleshooting

| Symptom | Fix |
|---------|-----|
| Login works locally but not from Netlify | Set `AUTOAPPLYER_CORS_ORIGINS` to the SPA origin; redeploy API; rebuild Netlify with `VITE_API_BASE_URL` pointing at Render. |
| 401 after login on split deploy | Cookie blocked: confirm HTTPS on both sides; check browser devtools → Application → Cookies. |
| Cold start timeout on Render | Normal on free tier; retry once. |

For VPS / Railway / Fly.io, see [`DEPLOYMENT.md`](../DEPLOYMENT.md).
